## Quick Setup

Run the AI setup command in your DevDoc project:

```bash
npx @brainfish-ai/devdoc ai
```

This creates:
- `CLAUDE.md` - Project context for Claude Code
- `.claude/skills/` - Custom slash commands
- `.cursor/rules/` - Cursor agent rules

## Documentation Types

The AI agents support three documentation types, generating appropriate structure for each:

| Type | Audience | Content Focus |
|------|----------|---------------|
| **internal** | Your team | Setup guides, architecture, contributing |
| **api** | External developers | API reference, authentication, SDKs |
| **product** | End users | Features, tutorials, FAQs |

### Configure via docs.json

Set your documentation type in `docs.json` to skip being asked each time:

```json
{
  "name": "My Project",
  "docType": "api",
  "navigation": { ... }
}
```

If not set, the AI will ask and then save your choice automatically.

## Key Capabilities

### Bootstrap Documentation

Generate complete documentation from your existing codebase:

<Tabs>
  <Tab title="Claude Code">
    ```
    /bootstrap-docs
    ```
  </Tab>
  <Tab title="Cursor">
    In Agent mode, ask:
    
    "Analyze this repo and generate initial DevDoc documentation"
  </Tab>
</Tabs>

The AI analyzes your README, package.json, source code, and API specs to create a complete documentation structure tailored to your doc type.

## Updating Skills

Keep your AI skills up to date with new features:

```bash
npx @brainfish-ai/devdoc ai --update
```

<Tip>
After updating, restart Claude Code or Cursor to load the new skills/rules.
</Tip>

## Available Commands

### Core Commands

| Command | Description |
|---------|-------------|
| `/create-doc` | Create new documentation pages interactively |
| `/update-doc` | Update docs by analyzing codebase changes |
| `/delete-doc` | Delete documentation pages with confirmation |
| `/commit-doc` | Commit documentation changes to git |

### Setup & Migration

| Command | Description |
|---------|-------------|
| `/bootstrap-docs` | Generate initial docs from codebase |
| `/migrate-docs` | Migrate from other platforms |
| `/import-api-spec` | Import OpenAPI/GraphQL specs |
| `/check-docs` | Quick health check |

## Working with Parent Directories

If your docs are in a subfolder (e.g., `my-project/docs/`), the AI automatically looks for source code in the parent directory.

The `CLAUDE.md` includes context about where to find source code:

```markdown
## Source Code Location

If this docs folder is inside a larger repository:
- Source code: `../src/` or `../lib/`
- Package config: `../package.json`
- README: `../README.md`
```
