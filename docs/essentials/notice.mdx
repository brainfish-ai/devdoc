title: "Notice"

Use notices to display important announcements, updates, or notifications across your entire documentation site. Notices appear at the top of every page, support Markdown formatting, and can be made dismissible.

## Configuration

Add a notice by including the `notice` property in your `docs.json`:

```json title="docs.json"
{
  "notice": {
    "content": "üöÄ Version 2.0 is now live! See our [changelog](/changelog) for details.",
    "dismissible": true
  }
}
```

</table>

## Markdown Support

Notice content supports Markdown formatting including:

- **Links**: `[link text](/path)` or `[external](https://example.com)`
- **Bold**: `**important**`
- **Italic**: `*emphasis*`
- **Code**: `` `inline code` ``

Example with rich formatting:

```json
{
  "notice": {
    "content": "üì¢ **Breaking change** in v3.0: The `oldMethod()` is deprecated. [See migration guide](/migrate).",
    "dismissible": true
  }
}
```

## Dismissible Behavior

When `dismissible` is set to `true`:

1. A close button (X) appears on the right side of the notice
2. Clicking the button hides the notice for the current session
3. The notice reappears when the user starts a new session
4. Dismissal state is stored in browser session storage

<Tip>
Use dismissible notices for non-critical announcements. For important notices that users must see, set `dismissible` to `false`.
</Tip>

## Best Practices

<Note>
Keep notice messages concise - aim for one line of text. Long messages may wrap on smaller screens.
</Note>

- **Be concise**: Keep messages short and actionable
- **Use emojis**: Add visual interest with relevant emojis
- **Include links**: Direct users to more information when relevant
- **Consider urgency**: Use non-dismissible notices only for critical information
- **Test on mobile**: Ensure notice content displays well on smaller screens
