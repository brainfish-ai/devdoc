Serve static assets from your documentation repository. All assets should be placed in the `assets/` folder at the root of your project.

<Note>
  **Convention:** All static files (images, videos, fonts, etc.) should go in the `assets/` folder. This is the recommended structure for DevDoc projects.
</Note>

## Project Structure

```text
/your-project
  ├── docs.json
  ├── theme.json
  ├── assets/                    # All static assets go here
  │   ├── logo/
  │   │   ├── light.svg         # Light mode logo
  │   │   └── dark.svg          # Dark mode logo
  │   ├── images/
  │   │   ├── hero.png
  │   │   └── screenshots/
  │   │       └── dashboard.png
  │   ├── videos/
  │   │   └── demo.mp4
  │   └── fonts/
  │       └── custom-font.woff2
  ├── index.mdx
  └── quickstart.mdx
```

Files in the `assets/` folder are automatically uploaded during deployment and served from your documentation domain.

<Tip>
  During `devdoc deploy`, binary assets (images, videos, fonts) are uploaded separately with progress tracking. Assets up to **25 MB** per file are supported.
</Tip>

## Referencing Assets

Assets are served from the `/assets/` path on your domain. Reference them in your MDX files:

```mdx
![Hero Image](/assets/images/hero.png)

<Image src="/assets/images/screenshot.png" alt="Screenshot" />
```

The `assets/` folder structure maps directly to URL paths:

| File Path | URL |
|-----------|-----|
| `assets/logo/light.svg` | `/assets/logo/light.svg` |
| `assets/images/hero.png` | `/assets/images/hero.png` |
| `assets/videos/demo.mp4` | `/assets/videos/demo.mp4` |

## Embedding Files

Embed files directly in your documentation with preview support.

### Images

Use the `Image` component for responsive images with optional captions:

<Image 
  src="/assets/images/screenshot.png" 
  alt="Self-service workflow"
  caption="Brainfish self-service documentation workflow"
/>

```jsx
<Image src="/assets/images/screenshot.png" alt="Dashboard screenshot" />

<Image 
  src="/assets/images/screenshot.png" 
  alt="Feature preview"
  caption="The new dashboard interface"
  width={600}
/>
```

### Videos

Embed videos with the `Video` component:

```jsx
<Video src="/assets/demo.mp4" />

<Video 
  src="/assets/tutorial.mp4"
  title="Getting Started Tutorial"
  autoplay={false}
  controls
/>
```

### PDF Documents

Embed PDF files with inline preview using the `PDF` component:

<PDF src="https://ontheline.trincoll.edu/images/bookdown/sample-local-pdf.pdf" title="Sample Document" height={400} />

```jsx
<PDF src="/assets/whitepaper.pdf" />

<PDF 
  src="/assets/guide.pdf" 
  height={600}
  title="User Guide"
/>
```

### Audio

Embed audio files with the `Audio` component:

```jsx
<Audio src="/assets/podcast.mp3" />

<Audio 
  src="/assets/interview.mp3"
  title="Interview with the Team"
/>
```

### Downloadable Files

Create download links for any file type using the `Download` component:

```jsx
<Download src="/assets/template.zip" label="Download Template" />

<Download 
  src="/assets/data.csv" 
  label="Export Data"
  icon="file-csv"
/>
```
