The `devdoc ai` command sets up AI agent configuration in your DevDoc project, enabling powerful documentation automation with Claude Code and Cursor.

## Usage

```bash
devdoc ai [options]
```

## Options

| Option | Description |
|--------|-------------|
| `-t, --tool <tool>` | AI tool to configure: `claude`, `cursor`, or `both` |

## With Flags

Skip the interactive prompt by specifying the tool:

```bash
# Claude Code only
devdoc ai --tool claude

# Cursor only
devdoc ai --tool cursor

# Both tools
devdoc ai --tool both
```

## Files Created

### Claude Code

| File | Purpose |
|------|---------|
| `CLAUDE.md` | Project context loaded into every conversation |
| `.claude/skills/bootstrap-docs/SKILL.md` | Generate docs from codebase |
| `.claude/skills/migrate-docs/SKILL.md` | Migrate from other platforms |
| `.claude/skills/import-api-spec/SKILL.md` | Import OpenAPI/GraphQL specs |
| `.claude/skills/sync-docs/SKILL.md` | Find and fix outdated docs |
| `.claude/skills/check-docs/SKILL.md` | Quick health check |
| `.claude/skills/create-doc-page/SKILL.md` | Create new MDX page |
| `.claude/skills/update-docs-json/SKILL.md` | Update navigation |
| `.claude/skills/generate-api-docs/SKILL.md` | Generate API reference |
| `.claude/skills/docs-from-code/SKILL.md` | Generate docs from code |

### Cursor

| File | Purpose |
|------|---------|
| `.cursor/rules/devdoc.mdc` | Core DevDoc writing rules |
| `.cursor/rules/devdoc-bootstrap.mdc` | Repository analysis rules |
| `.cursor/rules/devdoc-migrate.mdc` | Migration rules |
| `.cursor/rules/devdoc-sync.mdc` | Documentation sync rules |

## Available Commands After Setup

### Claude Code Skills

| Command | Description |
|---------|-------------|
| `/bootstrap-docs` | Analyze repo and generate initial documentation |
| `/migrate-docs` | Migrate from Mintlify, Docusaurus, GitBook, etc. |
| `/import-api-spec` | Import OpenAPI, GraphQL, or AsyncAPI specs |
| `/sync-docs` | Find and fix outdated documentation |
| `/check-docs` | Quick health check without changes |
| `/create-doc-page` | Create a new documentation page |
| `/update-docs-json` | Add pages to navigation |
| `/generate-api-docs` | Generate API documentation |
| `/docs-from-code` | Generate docs from specific code files |

### Cursor Prompts

In Agent mode, try these prompts:

- "Analyze this repo and generate initial documentation"
- "Migrate my Mintlify docs to DevDoc format"
- "Check my docs for outdated content"
- "Create a new guide about authentication"
- "Add this page to the navigation"

## Example Workflow

<Steps>
  <Step title="Set up AI agents">
    ```bash
    devdoc ai
    ```
  </Step>
  
  <Step title="Bootstrap documentation">
    In Claude Code:
    ```
    /bootstrap-docs
    ```
    
    Or in Cursor Agent mode:
    "Generate initial documentation from this repo"
  </Step>
  
  <Step title="Preview and refine">
    ```bash
    devdoc dev
    ```
    
    Review generated docs and make adjustments.
  </Step>
  
  <Step title="Keep docs in sync">
    After code changes, run:
    ```
    /sync-docs
    ```
  </Step>
</Steps>
