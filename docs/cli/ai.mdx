The `devdoc ai` command sets up AI agent configuration in your DevDoc project, enabling powerful documentation automation with Claude Code and Cursor.

## Usage

```bash
npx @brainfish-ai/devdoc ai [options]
```

## Options

| Option | Description |
|--------|-------------|
| `-t, --tool <tool>` | AI tool to configure: `claude`, `cursor`, or `both` |
| `--update` | Update existing skills/rules to latest version |

## Installing Skills

Running without options prompts you to select your AI tool:

```bash
$ npx @brainfish-ai/devdoc ai

DevDoc AI Agent Setup

Which AI tool do you use?

  1. Both - Claude Code and Cursor
  2. Claude Code - Skills and CLAUDE.md
  3. Cursor - Rules (.cursor/rules/)

Enter number: 1

Setting up AI agent configuration...

✓ Created CLAUDE.md
✓ Created .claude/skills/bootstrap-docs/SKILL.md
✓ Created .claude/skills/migrate-docs/SKILL.md
...

AI agent configuration complete!
```

## Updating Skills

Update your skills to the latest version by running the command again:

```bash
npx @brainfish-ai/devdoc ai
# or explicitly:
npx @brainfish-ai/devdoc ai --update
```

The command auto-detects existing installations and shows what's updated:

```bash
$ npx @brainfish-ai/devdoc ai

DevDoc AI Agent Update

Updating AI agent configuration...

✓ Updated CLAUDE.md
✓ Updated .claude/skills/bootstrap-docs/SKILL.md
✓ Updated .cursor/rules/devdoc-bootstrap.mdc
...

AI agent configuration updated! (10 files)

Tip: Restart Claude Code or Cursor to load the updated skills/rules.
```

<Warning>
After updating, restart Claude Code or Cursor to load the new skills/rules.
</Warning>

## With Flags

Skip the interactive prompt by specifying the tool:

```bash
# Claude Code only
npx @brainfish-ai/devdoc ai --tool claude

# Cursor only
npx @brainfish-ai/devdoc ai --tool cursor

# Both tools
npx @brainfish-ai/devdoc ai --tool both

# Update specific tool
npx @brainfish-ai/devdoc ai --tool claude --update
```

## Whisk Theme

The `whisk-theme` skill analyzes your codebase to extract brand colors, fonts, and logos, then generates a custom theme for your documentation.

**What it scans:**
- Tailwind config (`theme.colors.primary`)
- CSS variables (`--primary`, `--brand-color`)
- UI frameworks (Chakra, MUI, shadcn)
- Logo files (SVG, PNG)
- Font configurations

**What it generates:**
- `theme.json` with brand colors
- `custom.css` with branded component styles
- Copies logo to docs folder

```bash
# In Claude Code
/whisk-theme

# In Cursor
"Whisk theme" or "Match my branding"
```

<Tip>
Run whisk-theme after bootstrap to make your docs match your product's look and feel.
</Tip>

    ```bash
    npx @brainfish-ai/devdoc dev
    ```
    
    Review generated docs and make adjustments.
  </Step>
  
  <Step title="Update skills periodically">
    ```bash
    npx @brainfish-ai/devdoc ai --update
    ```
    
    Keep skills updated for new features and improvements.
  </Step>
</Steps>
