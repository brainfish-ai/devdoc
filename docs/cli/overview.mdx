The DevDoc CLI helps you develop, build, and deploy your documentation from the command line.

## Usage

Use the CLI with npx (recommended - no installation required):

```bash
npx @brainfish-ai/devdoc <command>
```

Or install globally:

```bash
npm install -g @brainfish-ai/devdoc
devdoc <command>
```

## Checking & Updating Version

Check which version you're running:

```bash
npx @brainfish-ai/devdoc --version
```

### Update to Latest

If you have an outdated version, use `@latest` to get the newest release:

```bash
npx @brainfish-ai/devdoc@latest <command>
```

### Clear Cache

npx caches packages locally. To force a fresh download:

```bash
# Clear the npx cache
npx clear-npx-cache

# Then run with @latest
npx @brainfish-ai/devdoc@latest --version
```

### Update Global Installation

If you installed globally:

```bash
npm update -g @brainfish-ai/devdoc
```

<Tip>
Use `@latest` when running commands to always get the newest features and fixes.
</Tip>

## Commands

| Command | Description |
|---------|-------------|
| `npx @brainfish-ai/devdoc create` | Create a new documentation project |
| `npx @brainfish-ai/devdoc dev` | Start development server with hot reload |
| `npx @brainfish-ai/devdoc build` | Build static site for production |
| `npx @brainfish-ai/devdoc deploy` | Deploy to DevDoc hosting |
| `npx @brainfish-ai/devdoc check` | Validate docs.json and content files |
| `npx @brainfish-ai/devdoc ai` | Set up AI agent configuration |
| `npx @brainfish-ai/devdoc keys` | Manage API keys for your project |
| `npx @brainfish-ai/devdoc domain` | Manage custom domains |
| `npx @brainfish-ai/devdoc whoami` | Show current project information |

## Global Options

These options work with all commands:

| Option | Description |
|--------|-------------|
| `--help` | Show help for a command |
| `--version` | Show CLI version |
| `--verbose` | Enable verbose logging |

## Configuration

The CLI looks for configuration in this order:

1. `docs.json` in the current directory
2. `DEVDOC_PROJECT_ROOT` environment variable
3. Parent directories (walks up the tree)

## Environment Variables

| Variable | Description |
|----------|-------------|
| `DEVDOC_API_KEY` | API key for deployments (CI/CD) |
| `DEVDOC_PROJECT_ROOT` | Path to project root |

## Quick Reference

```bash
# Create a new project
npx @brainfish-ai/devdoc@latest create my-docs

# Create with specific template and doc type
npx @brainfish-ai/devdoc@latest create my-docs --template openapi --doc-type public

# Create internal docs (no subdomain needed)
npx @brainfish-ai/devdoc@latest create my-docs --doc-type internal

# Start local development
npx @brainfish-ai/devdoc@latest dev

# Deploy to production
npx @brainfish-ai/devdoc@latest deploy

# Check for errors
npx @brainfish-ai/devdoc@latest check

# Set up or update AI agents
npx @brainfish-ai/devdoc@latest ai

# View project info
npx @brainfish-ai/devdoc@latest whoami

# Regenerate API key
npx @brainfish-ai/devdoc@latest keys regenerate

# Add custom domain
npx @brainfish-ai/devdoc@latest domain add docs.example.com

# Check domain status
npx @brainfish-ai/devdoc@latest domain status
```
