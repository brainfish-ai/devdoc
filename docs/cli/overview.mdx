The DevDoc CLI helps you develop, build, and deploy your documentation from the command line.

## Usage

**Create a new project** with npx:

```bash
npx @brainfish-ai/devdoc create my-docs
```

**Inside your project**, use the CLI directly or npm scripts:

```bash
# Using devdoc CLI
devdoc dev
devdoc deploy

# Or using npm scripts (recommended)
npm run dev
npm run deploy
```

<Tip>
When you create a project with `devdoc create`, the CLI is installed as a local dependency. Use `npm run` commands for the best experience.
</Tip>

## Checking & Updating Version

Check which version you're running:

```bash
devdoc --version
```

### Update to Latest

Inside your project, update devdoc to the latest version:

```bash
npm run upgrade
```

Or manually:

```bash
npm update @brainfish-ai/devdoc
```

## Commands

| Command | npm Script | Description |
|---------|------------|-------------|
| `devdoc dev` | `npm run dev` | Start development server with hot reload |
| `devdoc build` | `npm run build` | Build static site for production |
| `devdoc start` | `npm run start` | Start production server |
| `devdoc deploy` | `npm run deploy` | Deploy to DevDoc hosting |
| `devdoc check` | `npm run check` | Validate docs.json and content files |
| `devdoc ai` | `npm run ai` | Set up AI agent configuration |
| `devdoc sdk` | - | Generate client SDKs from OpenAPI spec |
| `devdoc whoami` | `npm run whoami` | Show current project information |
| `devdoc keys` | - | Manage API keys for your project |
| `devdoc domain` | - | Manage custom domains |

<Note>
Use `npx @brainfish-ai/devdoc create` only for creating new projects. Once inside a project, use `devdoc` or `npm run` commands.
</Note>

## Global Options

These options work with all commands:

| Option | Description |
|--------|-------------|
| `--help` | Show help for a command |
| `--version` | Show CLI version |
| `--verbose` | Enable verbose logging |

## Configuration

The CLI looks for configuration in this order:

1. `docs.json` in the current directory
2. `DEVDOC_PROJECT_ROOT` environment variable
3. Parent directories (walks up the tree)

## Environment Variables

| Variable | Description |
|----------|-------------|
| `DEVDOC_API_KEY` | API key for deployments (CI/CD) |
| `DEVDOC_PROJECT_ROOT` | Path to project root |

## Quick Reference

```bash
# Create a new project (use npx)
npx @brainfish-ai/devdoc create my-docs
npx @brainfish-ai/devdoc create my-docs --template openapi --doc-type public
npx @brainfish-ai/devdoc create my-docs --doc-type internal

# Inside your project (use devdoc or npm run)
cd my-docs

# Start local development
npm run dev          # or: devdoc dev

# Deploy to production
npm run deploy       # or: devdoc deploy

# Check for errors
npm run check        # or: devdoc check

# Set up or update AI agents
npm run ai           # or: devdoc ai

# View project info
npm run whoami       # or: devdoc whoami

# Update devdoc to latest
npm run upgrade

# Regenerate API key
devdoc keys regenerate

# Add custom domain
devdoc domain add docs.example.com

# Check domain status
devdoc domain status

# SDK generation
devdoc sdk init           # Initialize SDK config
devdoc sdk generate       # Generate all SDKs
devdoc sdk generate --lang typescript  # Generate specific language
devdoc sdk validate       # Validate OpenAPI spec
devdoc sdk list           # List available generators
```
