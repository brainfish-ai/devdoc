The `devdoc create` command scaffolds a new documentation project with your choice of template.

## Usage

```bash
npx @brainfish-ai/devdoc create [project-directory] [options]
```

## Options

| Option | Description |
|--------|-------------|
| `-t, --template <type>` | Template to use: `basic`, `openapi`, or `graphql` |
| `-d, --doc-type <type>` | Documentation type: `public` or `internal` |
| `-s, --subdomain <name>` | Subdomain for your docs (public only) |
| `--no-git` | Skip git initialization |
| `--no-install` | Skip dependency installation |
| `-h, --help` | Show help |

## Documentation Types

Choose the type that matches your use case:

| Type | Description | Subdomain | Deploy |
|------|-------------|-----------|--------|
| `public` | User-facing docs with custom subdomain and hosting | Yes | Yes |
| `internal` | Developer docs for local/private use | No | No |

<Tip>
Choose **internal** for team documentation, READMEs, or private wikis that don't need public hosting.
</Tip>

### With Project Name

Specify the project directory, then select a template:

```bash
npx @brainfish-ai/devdoc create my-docs
```

### Fully Specified

Skip all prompts:

```bash
# Basic template
npx @brainfish-ai/devdoc create my-docs --template basic

# OpenAPI template
npx @brainfish-ai/devdoc create my-docs --template openapi

# GraphQL template
npx @brainfish-ai/devdoc create my-docs --template graphql
```

### Skip Git and Install

For CI environments or manual setup:

```bash
npx @brainfish-ai/devdoc create my-docs --template basic --no-git --no-install
```

## After Creating

Once your project is created:

<Steps>
  <Step title="Navigate to your project">
    ```bash
    cd my-docs
    ```
  </Step>
  
  <Step title="Start development server">
    ```bash
    npm run dev
    # or
    npx @brainfish-ai/devdoc dev
    ```

    Your browser will automatically open to [http://localhost:3333](http://localhost:3333) with your docs.
  </Step>
</Steps>
