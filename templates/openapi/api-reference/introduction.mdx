Welcome to the REST API documentation. This API follows RESTful conventions and uses standard HTTP methods.

## Base URL

All API requests should be made to:

```
https://api.example.com/v1
```

## Request Format

The API accepts JSON-encoded request bodies and returns JSON-encoded responses.

```bash
curl -X POST https://api.example.com/v1/users \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"name": "John Doe", "email": "john@example.com"}'
```

## Response Format

All responses include a consistent structure:

```json
{
  "data": {
    // Response data
  },
  "meta": {
    "request_id": "req_abc123"
  }
}
```

## Pagination

List endpoints support pagination using `limit` and `offset` parameters:

```bash
GET /v1/users?limit=20&offset=40
```

The response includes pagination metadata:

```json
{
  "data": [...],
  "meta": {
    "total": 100,
    "limit": 20,
    "offset": 40
  }
}
```
